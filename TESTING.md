# Brikko Shield — Manual Testing Checklist (v0.2.0)

Расширение plain JS, без unit-тестов в репо. Этот чек-лист — обязательный smoke-test перед каждым релизом и после каждого редизайна Anthropic / OpenAI (селекторы могут устареть).

## Подготовка

1. Получите API-ключ: `https://brikko.ru/app` → API-ключи → Создать.
2. Откройте `chrome://extensions` (или `edge://extensions`).
3. Включите **Developer mode**.
4. **Load unpacked** → выберите папку `brikko-shield/`.
5. Откройте popup → вставьте API-ключ → **Сохранить**.

## 1. Установка и базовое здоровье

- [ ] В `chrome://extensions` нет красных ошибок под карточкой расширения.
- [ ] «service worker» под карточкой кликабельный, открывает DevTools без `Uncaught` в консоли.
- [ ] Иконка щита появилась в панели расширений.
- [ ] **Manifest V3 валидный** — `manifest_version: 3`, `host_permissions` включают `claude.ai`, `chatgpt.com`, `chat.openai.com`, `api.brikko.ru`.

## 2. claude.ai — V1 manual button (regression)

- [ ] Открыть `https://claude.ai`. В правом нижнем углу поля ввода появляется кнопка **«🛡 Защитить»**.
- [ ] Console на странице чистая (0 ошибок от расширения).
- [ ] Ввести: `Подготовь email клиенту Иванову Петру, ИНН 7707083893, тел +7 999 123-45-67.`
- [ ] Нажать **«🛡 Защитить»** — текст в поле меняется на маскированный (`<NAME_1>`, `<INN_1>`, `<PHONE_1>`).
- [ ] Бейдж справа сверху мелькает «Маскируем...» → «✓ ПД защищены».
- [ ] Самостоятельно нажать **Send** → Claude отвечает.
- [ ] После завершения стрима плейсхолдеры в ответе **подменяются** на оригинальные значения.

## 3. chat.openai.com / chatgpt.com — V2 manual button

- [ ] Открыть `https://chatgpt.com`. Кнопка **«🛡 Защитить»** появляется над `<textarea>`.
- [ ] Console чистая.
- [ ] Ввести тот же текст с ПД → нажать **«🛡 Защитить»** → маскирование работает.
- [ ] **Send** → ответ ChatGPT → плейсхолдеры подменяются обратно.
- [ ] Повторить на `https://chat.openai.com` — поведение идентичное (тот же домен у OpenAI).

## 4. Auto-intercept Enter mode

- [ ] Open popup → включить toggle **«Автоматическая защита (Enter)»**.
- [ ] На claude.ai: ввести текст с ПД → **Enter** (без Shift) → текст ДОЛЖЕН подмениться на маскированный → отправиться сам.
- [ ] Бейдж: «Маскируем...» → «✓ ПД защищены».
- [ ] **Shift+Enter** оставляет перенос строки, не триггерит маскирование.
- [ ] То же самое на chatgpt.com — поведение идентичное.
- [ ] Выключить toggle → Enter снова отправляет plaintext (поведение V1).

## 5. PII-less prompt — UX без шума

- [ ] При **включённом** auto-intercept ввести: `Сколько будет дважды два?` → **Enter**.
- [ ] Текст НЕ должен поменяться, бейдж НЕ должен появиться.
- [ ] Запрос отправляется как есть, без вмешательства.

## 6. Fail-safe: api.brikko.ru недоступен

- [ ] **DevTools → Network → Throttling → Offline** (на странице чата).
- [ ] При **включённом** auto-intercept ввести prompt с ПД → **Enter**.
- [ ] Ожидаем: бейдж «✗ Ошибка защиты» (красный), prompt **НЕ отправляется** в LLM.
- [ ] Plaintext остаётся в поле ввода — пользователь видит, что нужно повторить.
- [ ] Console на странице: предупреждение опционально, но `console.error` от расширения быть не должно.

## 7. Options page (`chrome://extensions` → Details → Extension options ИЛИ popup → «Настройки»)

- [ ] Страница открывается в новой вкладке.
- [ ] Все секции рендерятся: API-ключ, Защита, Сайты, Статистика.
- [ ] API-status показывает «Защита активна» если ключ валидный.
- [ ] Toggle **«claude.ai»** выкл → перезагрузить вкладку Claude → кнопка «🛡 Защитить» НЕ появляется.
- [ ] Toggle **«chatgpt.com»** выкл → перезагрузить вкладку ChatGPT → кнопка не появляется.
- [ ] Включить обратно → кнопка возвращается без перезагрузки расширения.
- [ ] Статистика: после 1-2 успешных маскирований счётчик «защищено сегодня» увеличивается.

## 8. Stream support (post-stream restore)

- [ ] Маскированный prompt отправлен → ответ начинает стримиться.
- [ ] Во время стрима в ответе видны плейсхолдеры (`<NAME_1>` и т.д.) — это контракт V2.
- [ ] Через ≤1 сек после конца стрима плейсхолдеры подменяются на оригиналы.
- [ ] Если оригинальный prompt был перефразирован моделью (например, Claude склонил `<NAME_1>` как `<NAME_1>'s`) — расширение корректно оставляет такие фрагменты как есть, не ломая markdown.

## 9. Per-tab isolation

- [ ] В одной вкладке Claude замаскировать prompt → mapping_id живёт только для этой вкладки.
- [ ] Открыть **новую** вкладку chatgpt.com → она не знает о mapping_id первой вкладки (это by design — изоляция).
- [ ] Закрыть вкладку Claude → её state в background SW удаляется (нет утечки).
- [ ] Перейти на `https://example.com` в той же вкладке (URL change) → state очищается.

## 10. Privacy / no telemetry

- [ ] DevTools → Network вкладки чата: единственные исходящие запросы расширения — `https://api.brikko.ru/v1/anonymize` и `https://api.brikko.ru/v1/restore`.
- [ ] Никаких analytics, Sentry, Mixpanel, Plausible — ноль внешних доменов.
- [ ] `manifest.json.host_permissions` содержит ровно 4 домена.

## Известные ограничения V2 (для V3)

- Авто-перехват Enter работает через `keydown` capture + `preventDefault`. Если сайт переключится на framework-уровневый submit (например, IME composition tricks), можно обходить — нужен hook `beforeinput` или фрейм-капчер на форме.
- Stream-finished детекция эвристическая (отсутствие `.result-streaming` / `[data-is-streaming="true"]`). При ре-стриминге (regenerate) restore сделается заново.
- Нет alignment-восстановления при перефразировании — restore консервативный (см. `buildPlaceholderMap` в `content/_shared.js`).
- `chrome.storage.sync` лимит 100 KB / 8 KB на ключ. Если статистика разрастётся — мигрировать на `chrome.storage.local` в V3.
