# Brikko Shield

**Браузерное расширение для защиты персональных данных при работе с LLM-чатами.**

Перед отправкой сообщения в ChatGPT или Claude расширение маскирует ИНН, СНИЛС, ФИО, телефоны, паспорта, e-mail, номера карт и пр. через [api.brikko.ru](https://brikko.ru). В ответе модели плейсхолдеры (`<NAME_1>`, `<INN_1>` и т.д.) автоматически восстанавливаются — вы читаете реальные имена, LLM обрабатывает только маски.

> **Status:** v0.2.0. Поддерживаются claude.ai, chat.openai.com, chatgpt.com.

## Установка (Chrome / Edge)

1. **Скачайте** репозиторий — кнопка **Code → Download ZIP** на GitHub, либо `git clone https://github.com/brikkoAI/brikko-shield.git`.
2. Распакуйте архив (если ZIP). Должна получиться папка `brikko-shield/` с `manifest.json` внутри.
3. Откройте Chrome → введите в адресной строке `chrome://extensions` (для Edge: `edge://extensions`).
4. Включите тумблер **«Режим разработчика»** (Developer mode) в правом верхнем углу.
5. Нажмите **«Загрузить распакованное расширение»** (Load unpacked) → выберите папку `brikko-shield/`.
6. Иконка щита появится в панели расширений. Закрепите её (значок-пазл → булавка).

> Расширение пока не опубликовано в Chrome Web Store — это произойдёт в v1.0 после длительного бета-теста.

## Настройка

1. Откройте `https://brikko.ru/app` → раздел **API-ключи** → создайте ключ для расширения.
2. Кликните иконку Brikko Shield в панели расширений.
3. Вставьте ключ в поле **API-ключ Brikko** → **Сохранить**.
4. Индикатор сменится на зелёный «Защита активна».
5. (Опционально) откройте **Настройки** для расширенного управления — per-site toggle, статистика.

## Как пользоваться

### Ручной режим (по умолчанию)

1. Откройте `https://claude.ai` или `https://chatgpt.com`.
2. Напишите сообщение с ПД как обычно — например:
   > _Подготовь email клиенту Иванову Петру, ИНН 7707083893, тел +7 999 123-45-67._
3. Нажмите **🛡 Защитить** — кнопка появляется над полем ввода.
4. Текст в поле подменится на маскированный:
   > _Подготовь email клиенту `<NAME_1>`, ИНН `<INN_1>`, тел `<PHONE_1>`._
5. Просмотрите результат и **сами нажмите Send** — LLM получит только маски.
6. Когда придёт ответ, расширение автоматически восстановит плейсхолдеры — вы увидите оригинальные имена и номера, провайдер модели — нет.

### Авто-режим (V2)

1. В popup включите toggle **«Автоматическая защита (Enter)»**.
2. Пишите как обычно. **Enter** теперь:
   - Отменяет «голую» отправку.
   - Маскирует текст через api.brikko.ru.
   - Программно нажимает Send — в LLM уходит уже маска.
3. **Shift+Enter** (перенос строки) работает как раньше.
4. Если в тексте **нет ПД**, расширение не вмешивается — никаких пустых бейджей и задержек.

> **Защита от утечки:** если api.brikko.ru недоступен, авто-режим **НЕ отправит** plaintext в LLM. Бейдж «✗ Ошибка защиты» — пользователь видит, что нужно повторить.

### Бейдж в углу

- **«Маскируем...»** (серый) — идёт запрос к api.brikko.ru.
- **«✓ ПД защищены»** (зелёный) — маскирование успешно, авто-исчезает через 4 сек.
- **«✗ Ошибка защиты»** (красный) — Brikko недоступен, или API-ключ отозван. Запрос **не отправлен**.

## Архитектура (v0.2.0)

```
chrome.storage.sync ──┐                         api.brikko.ru
                      ↓                              ▲
              ┌──────────────────┐                   │ POST /v1/anonymize
              │ background SW    │ ──────────────────┤ POST /v1/restore
              │ + per-tab state  │                   │
              └────────▲─────────┘
                       │ chrome.runtime.sendMessage
                       │
              ┌────────┴─────────┐
              │ content/_shared  │ ← общий runtime: badge, alignment,
              │ .js              │   anonymize/restore, auto-intercept
              └────────▲─────────┘
                       │
        ┌──────────────┴───────────────┐
        │                              │
┌───────────────┐              ┌───────────────────┐
│ claude.ai.js  │              │ chatgpt.com.js    │ ← site adapters:
│ (selectors)   │              │ (selectors)       │   selectors + send-trigger
└───────────────┘              └───────────────────┘
```

**Manifest V3**, plain JS, no build step. Никакого TypeScript, Vite, бандлеров. ~600 строк JS суммарно.

### Smoke-test перед каждым релизом

См. полный чек-лист: [`TESTING.md`](./TESTING.md). Минимум:

1. claude.ai — ручная кнопка работает (V1 regression).
2. chatgpt.com — ручная кнопка работает.
3. Auto-intercept Enter — Enter маскирует и отправляет.
4. PII-less prompt — расширение пропускает без вмешательства.
5. api.brikko.ru down → запрос НЕ уходит в LLM.

## Известные ограничения v0.2.0

- **Стриминг restore не потоковый.** Восстановление происходит после конца стрима (≤1 сек тишины). Во время стрима пользователь видит плейсхолдеры — это сознательный контракт: пользователь убеждается, что маскирование работает.
- **Alignment при перефразировании.** Если модель *перефразировала* плейсхолдер (например, склонила `<NAME_1>` → `<NAME_1>'s`), расширение оставит этот фрагмент как есть.
- **Без Studio sidecar / regex-fallback.** Только Gateway. Если api.brikko.ru недоступен — маскирование не работает (badge «Ошибка защиты»).
- **Без telemetry / audit log** — приватный продукт не должен сам утекать данные. Аудит-лог в popup — V3.
- **Selectors могут устареть.** ChatGPT и Claude часто меняют DOM. См. fallback-chains в `content/claude.ai.js` и `content/chatgpt.com.js` — там по 4 селектора на каждый элемент.
- **`chrome.storage.sync` лимит** 100 KB / 8 KB на ключ. Статистика «защищено сегодня» хранится одним числом, проблем нет до миллионов запросов.

## Безопасность и приватность

- Расширение **не отправляет** никакие данные никуда, кроме `api.brikko.ru` (см. `host_permissions` в `manifest.json`).
- API-ключ хранится в `chrome.storage.sync` — синхронизируется между вашими устройствами Chrome, шифруется в облаке Google, **не виден** вкладкам и другим расширениям.
- Маскирование работает по принципу _client-side gating_: LLM никогда не видит исходный текст, только маски.
- Mapping (соответствие плейсхолдер → исходное значение) живёт **в памяти background SW**, не сохраняется на диск, очищается при навигации вкладки на неподдерживаемый домен и при закрытии вкладки.
- **Никакой telemetry**, никаких analytics-доменов в `host_permissions`. Если кто-то найдёт исходящий трафик в чужой домен — это баг, открывайте issue.
