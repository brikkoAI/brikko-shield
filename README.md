# Brikko Shield

**Брaузерное расширение для защиты персональных данных при работе с Claude.ai.**

Перед отправкой сообщения в Claude расширение маскирует ИНН, СНИЛС, ФИО, телефоны, паспорта, e-mail, номера карт и пр. через [api.brikko.ru](https://brikko.ru). В ответе модели плейсхолдеры (`<NAME_1>`, `<INN_1>` и т.д.) автоматически восстанавливаются — вы читаете реальные имена, Claude обрабатывает только маски.

> **Status:** v0.1.0 MVP. Только Claude.ai. ChatGPT, GigaChat и Yandex Alice — в v0.2.

---

## Установка (Chrome / Edge)

1. **Скачайте** репозиторий — кнопка **Code → Download ZIP** на GitHub, либо `git clone`.
2. Распакуйте архив (если ZIP). Должна получиться папка `brikko-shield/` с `manifest.json` внутри.
3. Откройте Chrome → введите в адресной строке `chrome://extensions` (для Edge: `edge://extensions`).
4. Включите тумблер **«Режим разработчика»** (Developer mode) в правом верхнем углу.
5. Нажмите **«Загрузить распакованное расширение»** (Load unpacked) → выберите папку `brikko-shield/`.
6. Иконка щита появится в панели расширений. Закрепите её (значок-пазл → булавка).

## Настройка

1. Откройте `https://brikko.ru/app` → раздел **API-ключи** → создайте ключ для расширения.
2. Кликните иконку Brikko Shield в панели расширений.
3. Вставьте ключ в поле **API-ключ Brikko** → **Сохранить**.
4. Индикатор сменится на зелёный «Защита активна».

## Как пользоваться

1. Откройте `https://claude.ai`.
2. Напишите сообщение с ПД как обычно — например:
   > _Подготовь email клиенту Иванову Петру, ИНН 7707083893, тел +7 999 123-45-67._
3. Нажмите **🛡 Защитить** — кнопка появляется над полем ввода Claude.
4. Текст в поле подменится на маскированный:
   > _Подготовь email клиенту `<NAME_1>`, ИНН `<INN_1>`, тел `<PHONE_1>`._
5. Просмотрите результат и **сами нажмите Send** — Claude получит только маски.
6. Когда придёт ответ, расширение автоматически восстановит плейсхолдеры — вы увидите оригинальные имена и номера, Anthropic — нет.

В правом верхнем углу страницы будет ненадолго появляться плашка **«✓ ПД защищены»**.

## Архитектура (MVP)

```
chrome.storage.sync ──┐                         api.brikko.ru
                      ↓                              ▲
              ┌──────────────────┐                   │ POST /v1/anonymize
              │ background SW    │ ──────────────────┤ POST /v1/restore
              │ (service-worker) │                   │
              └────────▲─────────┘
                       │ chrome.runtime.sendMessage
                       │
              ┌────────┴─────────┐
              │ content script   │   on click "Защитить":
              │ (claude.ai.js)   │     - read text from ProseMirror
              └────────▲─────────┘     - replace with masked
                       │                - track mapping_id
                       │              on assistant response:
                       │                - MutationObserver → restore
                       ▼
                claude.ai DOM
```

**Manifest V3**, plain JS, no build step. Никакого TypeScript, Vite, бандлеров.

## Разработка

```bash
git clone https://github.com/brikkoAI/brikko-shield.git
cd brikko-shield
# нет npm install — это plain JS
```

Загрузите распакованное расширение (см. **Установка**), вносите правки, нажимайте кнопку «Reload» рядом с расширением в `chrome://extensions`, чтобы перезагрузить.

### Тестирование вручную

1. **Manifest валидный** — при загрузке нет красных ошибок в `chrome://extensions`.
2. **Service worker запускается** — кликните «service worker» под карточкой расширения → откроется DevTools, console чистая.
3. **API-ключ сохраняется** — popup → ввести ключ → закрыть/открыть popup → ключ остался.
4. **Кнопка появляется** на claude.ai — поверх поля ввода.
5. **Маскирование работает** — нажать «🛡 Защитить» → текст в поле меняется → бейдж становится зелёным.
6. **Восстановление работает** — после ответа Claude плейсхолдеры в ответе подменяются на оригиналы.

## Известные ограничения v0.1.0 (MVP)

- **Только Claude.ai.** ChatGPT, GigaChat — в v0.2.
- **Ручной режим.** Маскирует только при нажатии «🛡 Защитить» (а не автоматически на Enter). Это сознательное решение MVP — авто-перехват ProseMirror Claude нестабилен между релизами Anthropic, и пользователю важно видеть, *что именно* отправляется.
- **Нет потокового восстановления.** Восстановление ответа происходит после того, как стриминг закончился (через ~600ms тишины). Во время стрима пользователь видит плейсхолдеры.
- **Нет alignment-восстановления при перефразировании.** Если Claude *перефразировал* плейсхолдер (например, склонил `<NAME_1>` как `<NAME_1>'s`), восстановление пропустит этот участок и оставит как есть.
- **Без Studio sidecar / regex-fallback.** Только Gateway. Если api.brikko.ru недоступен — маскирование не работает (badge «Ошибка защиты»).
- **Без telemetry / audit log / quota UI** — V2.
- **Selectors могут устареть** — если Anthropic обновит DOM, кнопка может не появиться. См. `INPUT_SELECTORS` в `content/claude.ai.js`.

## Безопасность и приватность

- Расширение **не отправляет** никакие данные никуда, кроме `api.brikko.ru` (см. `host_permissions` в `manifest.json`).
- API-ключ хранится в `chrome.storage.sync` — синхронизируется между вашими устройствами Chrome, шифруется в облаке Google, **не виден** вкладкам и другим расширениям.
- Маскирование работает по принципу _client-side gating_: Claude никогда не видит исходный текст, только маски.
- Mapping (соответствие плейсхолдер → исходное значение) **не сохраняется** в браузере между сессиями — только в памяти текущей вкладки.
